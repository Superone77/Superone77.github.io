第一章：AI Agent 概览 #

理解 AI Agent 的核心概念、发展历程、分类体系和开发挑战

1.1 什么是 AI Agent #

1.1.1 Agent 的定义 #

学术定义（Russell & Norvig） AI Agent 是一个能够感知环境、做出决策并执行行动的系统，其目标是最大化某个性能指标（奖励函数）。

Agent = Architecture + Program

工程定义（2025 年最新） 在实际工程中，AI Agent 通常被定义为：

  1. 自主决策系统

    • 能够感知环境状态（用户输入、系统状态、外部数据）
    • 能够制定行动计划（基于规划、推理或学习）
    • 能够执行具体行动（调用工具、发送消息、更新状态）
    • 能够从经验中学习（通过反馈优化策略）

  2. 基于 LLM 的智能体（IBM 2024）

    • 使用大语言模型作为核心推理引擎
    • 使用工具调用（Tool Calling）扩展能力
    • 使用工作流管理（Workflow Management）进行复杂任务规划
    • 使用记忆系统（Memory System）存储历史信息

2025 年新增特征（基于 Agentic AI 设计模式）

  • 工具调用能力：Agent 能够调用外部工具完成任务
  • 工作流管理能力：Agent 能够管理复杂的工作流
  • 长期规划能力：Agent 能够进行长期规划和状态管理
  • 多 Agent 协作能力：Agent 能够与其他 Agent 协作
  • 反思与自学习能力：Agent 能够从自己的行为中学习

代码示例（Agent 基础结构）

from typing import Dict, Any, List
import openai

class AIAgent:
    def __init__(self, model="gpt-4"):
        self.client = openai.OpenAI(api_key="your-api-key")
        self.model = model

    def perceive(self, user_input: str) -> Dict[str, Any]:
        """感知用户输入"""
        return {
            "type": "text",
            "content": user_input
        }

    def plan(self, objective: str, context: str = "") -> List[Dict[str, Any]]:
        """规划行动计划（2025 年新增：工作流管理）"""
        prompt = f"""
        目标：{objective}
        上下文：{context}

        请使用思维链（Chain of Thought）进行规划：

        步骤 1：分析目标，理解需求
        步骤 2：将目标分解为子任务
        步骤 3：确定子任务的执行顺序和依赖关系
        步骤 4：为每个子任务选择合适的工具和参数
        步骤 5：输出完整的行动计划
        """
        response = self.client.chat.completions.create(
            model=self.model,
            messages=[{"role": "user", "content": prompt}],
            temperature=0.3
        )
        return response.choices[0].message.content

    def act(self, action: Dict[str, Any]) -> Dict[str, Any]:
        """执行行动（2025 年新增：工具调用）"""
        action_type = action.get("type")

        if action_type == "tool_call":
            result = self._execute_tool_call(action)
        elif action_type == "message_send":
            result = self._send_message(action)
        elif action_type == "state_update":
            result = self._update_state(action)
        else:
            result = {"success": False, "error": f"Unknown action type: {action_type}"}

        return result

    def _execute_tool_call(self, action: Dict[str, Any]) -> Dict[str, Any]:
        """执行工具调用（Function Calling）"""
        tool_name = action.get("tool")
        params = action.get("params", {})
        print(f"Calling tool: {tool_name} with params: {params}")

        # 这里实现实际的工具调用逻辑
        # 例如：调用 API、执行函数、查询数据库等

        return {
            "success": True,
            "tool": tool_name,
            "result": "Tool execution result"
        }

    def _send_message(self, action: Dict[str, Any]) -> Dict[str, Any]:
        """发送消息"""
        recipient = action.get("recipient")
        message = action.get("message")
        print(f"Sending message to {recipient}: {message}")

        return {
            "success": True,
            "message": message,
            "recipient": recipient
        }

    def _update_state(self, action: Dict[str, Any]) -> Dict[str, Any]:
        """更新状态（2025 年新增：长期记忆）"""
        state_type = action.get("state_type")
        data = action.get("data")
        print(f"Updating {state_type} state: {data}")

        # 这里实现实际的状态更新逻辑
        # 例如：更新数据库、写入文件、更新内存等

        return {
            "success": True,
            "state_type": state_type,
            "updated_data": data
        }

# 使用示例
agent = AIAgent()

# 感知
perception = agent.perceive("帮我规划一个北京到上海的旅行")
print("Perception:", perception)

# 规划

# 执行行动

    "type": "tool_call",
    "tool": "flight_search_api",
    "params": {
        "origin": "北京",
        "destination": "上海",
        "date": "2025-03-20"
    }
}

print("Action Result:", result)

1.2 Agent 发展历程 #

1.2.1 早期 Agent 系统（符号主义 Agent，1950s-1980s） #

核心思想 基于符号推理和知识工程的 Agent 系统。

代表系统

  1. GOFAI（Good Old-Fashioned AI）

    • 技术：基于符号逻辑的 AI 系统
    • 特点：符号推理、逻辑规则
    • 典型案例：简单的基于规则的 AI 系统

  2. Expert Systems（专家系统）

    • 技术：基于领域专家知识构建的系统
    • 特点：知识库（KB）、推理引擎、规则库
    • 典型案例：MYCIN（医疗诊断）、DENDRAL（化学分析）

  3. Shakey the Robot（1969）

    • 技术：第一个在复杂环境中自主移动的机器人
    • 特点：符号推理、感知-规划-执行循环
    • 典型案例：早期机器人控制

技术特点

  • 符号推理（Symbolic Reasoning）：使用逻辑规则进行推理
  • 知识表示（Knowledge Representation）：使用知识库（KB）表示领域知识
  • 推理引擎（Reasoning Engine）：使用推理引擎（如 MYCIN、DENDRAL）进行推理

优势

  • 可解释性强：推理过程透明
  • 知识可维护：知识库可以独立更新
  • 逻辑精确：推理结果可靠

劣势

  • 知识获取瓶颈：需要人工构建知识库
  • 泛化能力差：难以处理不确定性
  • 适应性差：难以适应环境变化

代码示例（基于规则的 Agent）

class RuleBasedAgent:
    def __init__(self):
        # 定义规则库
        self.rules = [
            {"if": "weather", "then": "query_weather"},
            {"if": "book_flight", "then": "book_flight"},
            {"if": "time", "then": "get_time"},
            {"if": "unknown", "then": "ask_human"}
        ]

    def decide_action(self, user_input: str) -> str:
        """决定行动（规则匹配）"""
        for rule in self.rules:
            if rule["if"] in user_input:
                return rule["then"]
        return "ask_human"

    def execute_action(self, action: str):
        """执行行动"""
        if action == "query_weather":
            print("查询天气 API")
        elif action == "book_flight":
            print("查询航班 API")
        elif action == "get_time":
            print("获取当前时间")
        elif action == "ask_human":
            print("请问需要什么帮助？")

# 使用示例
agent = RuleBasedAgent()
user_input = "北京明天天气"
action = agent.decide_action(user_input)
agent.execute_action(action)  # 输出：查询天气 API

1.2.2 强化学习 Agent（1990s-2010s） #

核心思想 基于强化学习的 Agent 系统，通过试错学习最优策略。

代表算法

  1. Q-Learning（1989）

    • 技术：基于值函数的强化学习算法
    • 特点：基于值函数的学习方法
    • 典型案例：简单的 Q-Learning 表格型 Agent

  2. DQN（Deep Q-Network，2013）

    • 技术：深度 Q-Network，结合深度学习和 Q-Learning
    • 特点：可以处理高维状态空间、端到端学习
    • 典型案例：Atari 游戏、机器人控制

  3. PPO（Proximal Policy Optimization，2017）

    • 技术：近端策略优化
    • 特点：训练稳定、样本效率高、适合连续动作空间
    • 典型案例：机器人控制、游戏 AI、推荐系统

技术特点

  • 状态空间（State Space）：环境状态的表示
  • 动作空间（Action Space）：Agent 可执行的动作
  • 奖励函数（Reward Function）：环境反馈的奖励信号
  • 策略函数（Policy Function）：从状态到动作的映射

优势

  • 自适应：能够适应环境变化
  • 学习能力：能够从经验中学习
  • 泛化能力强：能够泛化到新状态

劣势

  • 样本效率低：需要大量样本训练
  • 奖励设计困难：奖励函数设计影响学习效果
  • 黑盒性：神经网络决策过程不透明

代码示例（Q-Learning Agent）

import numpy as np

class QLearningAgent:
    def __init__(self, state_size: int, action_size: int, learning_rate=0.1, discount=0.95):
        self.state_size = state_size
        self.action_size = action_size
        self.learning_rate = learning_rate
        self.discount = discount
        # Q-Table：状态-动作的价值函数
        self.q_table = np.zeros((state_size, action_size))

    def choose_action(self, state: int) -> int:
        """选择行动（ε-greedy 策略）"""
        if np.random.random() < 0.1:
            # 探索：随机选择
            return np.random.randint(self.action_size)
        else:
            # 利用：选择最优行动
            return np.argmax(self.q_table[state])

    def update_q_table(self, state: int, action: int, reward: float, next_state: int):
        """Q-Learning 更新公式"""
        best_next_action = np.argmax(self.q_table[next_state])
        td_target = reward + self.discount * self.q_table[next_state][best_next_action]
        td_error = td_target - self.q_table[state][action]
        self.q_table[state][action] += self.learning_rate * td_error

    def run_episode(self, env, max_steps: int = 1000):
        """运行一个回合"""
        state = env.reset()
        total_reward = 0

        for step in range(max_steps):
            # 选择行动
            action = self.choose_action(state)

            # 执行行动
            next_state, reward, done, _ = env.step(action)

            # 学习
            self.update_q_table(state, action, reward, next_state)

            # 更新状态
            state = next_state
            total_reward += reward

            if done:
                break

        return total_reward

class SimpleEnv:
    def __init__(self):
        self.state = 0
        self.n_states = 10
        self.n_actions = 2  # 0: 左, 1: 右

    def reset(self):
        self.state = 0
        return self.state

    def step(self, action):
        if action == 0:  # 左
            self.state = max(0, self.state - 1)
        else:  # 右
            self.state = min(self.n_states - 1, self.state + 1)

        reward = 1 if self.state == self.n_states - 1 else -0.1
        done = self.state == self.n_states - 1
        return self.state, reward, done, {}

# 创建并训练 Agent
env = SimpleEnv()
agent = QLearningAgent(state_size=env.n_states, action_size=env.n_actions)
rewards = agent.run_episode(env, n_episodes=1000)

import openai

class LLMPlanner:
    def __init__(self, model="gpt-4"):
        self.client = openai.OpenAI(api_key="your-api-key")
        self.model = model

    def plan(self, objective: str, context: str = "") -> Dict[str, Any]:
        """使用 CoT 进行规划（2025 年新增：ReAct）"""
        prompt = f"""
        目标：{objective}
        上下文：{context}

        请使用 ReAct（推理+行动循环）进行规划：

        循环（最多 10 次）：
        1. 思考（Thought）：分析当前状态和目标
        2. 行动（Action）：选择要执行的行动
        3. 观察（Observation）：观察行动结果
        4. 评估：评估是否达到目标
        5. 如果达到目标，退出循环；否则继续

        输出格式：
        {{
            "thought": "思考内容",
            "action": {{"tool": "工具名", "params": {"参数名": "参数值"}}},
            "observation": "观察结果",
            "done": true/false
        }}
        """
        response = self.client.chat.completions.create(
            model=self.model,
            messages=[{"role": "user", "content": prompt}],
            temperature=0.3
        )
        return response.choices[0].message.content

    def replan(self, objective: str, current_state: str, error: str) -> Dict[str, Any]:
        """根据当前状态和错误重新规划"""
        prompt = f"""
        目标：{objective}
        当前状态：{current_state}
        错误：{error}

        请根据当前状态和错误重新规划：
        1. 分析错误原因
        2. 调整计划
        3. 输出新的行动计划
        """
        response = self.client.chat.completions.create(
            model=self.model,
            messages=[{"role": "user", "content": prompt}],
            temperature=0.3
        )
        return response.choices[0].message.content

import chromadb
from typing import List, Dict, Any
import openai

class MemoryModule:
    def __init__(self, collection_name: str = "agent_memory"):
        # 初始化 ChromaDB
        self.client = chromadb.PersistentClient(path="./chroma_db")
        self.collection = self.client.get_or_create_collection(
            name=collection_name
        )
        self.openai_client = openai.OpenAI(api_key="your-api-key")

    def add(self, text: str, metadata: Dict[str, Any] = None):
        """添加记忆项（2025 年新增：向量检索）"""
        # 获取文本向量
        embedding = self._get_embedding(text)

        # 添加到向量数据库
        self.collection.add(
            documents=[text],
            embeddings=[embedding],
            metadatas=[metadata or {}]
        )

    def search(self, query: str, n_results: int = 5) -> List[Dict[str, Any]]:
        """搜索记忆（基于语义检索）"""
        # 获取查询向量
        query_embedding = self._get_embedding(query)

        # 搜索向量数据库
        results = self.collection.query(
            query_embeddings=[query_embedding],
            n_results=n_results
        )

        # 格式化结果
        formatted_results = []
        for i in range(len(results["ids"][0])):
            formatted_results.append({
                "id": results["ids"][0][i],
                "text": results["documents"][0][i],
                "metadata": results["metadatas"][0][i],
                "distance": results["distances"][0][i]
            })

        return formatted_results

    def _get_embedding(self, text: str) -> List[float]:
        """获取文本向量"""
        response = self.openai_client.embeddings.create(
            model="text-embedding-3-small",
            input=text
        )
        return response.data[0].embedding

                    "content": f"用户输入：{user_input}。请选择合适的工具并调用。"
                }
            ],
            temperature=0.3,
            tools=[{
                "type": "function",
                "function": {
                    "name": "call_tool",
                    "description": "调用指定的工具",
                    "parameters": {
                        "type": "object",
                        "properties": {
                            "tool_name": {
                                "type": "string",
                                "enum": list(available_tools.keys())
                            },
                            "params": {
                                "type": "object",
                                "properties": {}
                            }
                        },
                        "required": ["tool_name"]
                    }
                }
            }]
        )

        return response.choices[0].message

# 使用示例
tool_module = ToolCallingModule()

# 用户输入
user_input = "帮我查询北京明天的天气"

# 决策工具并调用
response = tool_module.decide_tool_and_call(user_input)
print("Response:", response)

1.4.4 安全与对齐 #

挑战描述 如何防止 Agent 产生有害行为，以及如何让 Agent 的行为符合人类价值观。

具体挑战

  1. 数据泄露

    • 如何防止敏感数据泄露

  2. 恶意攻击

    • 如何防止对抗攻击、提示词注入、越狱攻击

  3. 滥用风险

    • 如何防止 Agent 滥用工具

  4. 对齐挑战

    • 如何让 Agent 的行为符合人类价值观

解决方案（基于最新研究）

  1. 输入过滤（Input Filtering）

    • 敏感词过滤：过滤敏感信息
    • 格式验证：验证输入格式
    • 长度限制：限制输入长度

  2. 输出过滤（Output Filtering）

    • 内容审核：审核输出内容
    • 敏感信息检测：检测敏感信息
    • 格式检查：检查输出格式

  3. Constitutional AI

    • 基于原则的 AI
    • 示例：Anthropic Claude
    • 优势：安全性强、对齐性好

  4. RLHF（Reinforcement Learning from Human Feedback）

    • 基于人类反馈的强化学习
    • 示例：ChatGPT
    • 优势：对齐人类偏好、行为符合价值观

代码示例（安全与对齐）

import openai

class SafeAIAgent:
    def __init__(self, model="gpt-4"):
        self.client = openai.OpenAI(api_key="your-api-key")
        self.model = model

    def safe_respond(self, user_input: str) -> str:
        """安全响应（2025 年新增：Constitutional AI）"""
        # 定义安全原则
        constitution = [
            "不提供有害或非法的建议",
            "不泄露用户隐私信息",
            "不鼓励危险行为",
            "提供有用、安全和诚实的回答"
        ]

        prompt = f"""
        用户输入：{user_input}

        安全原则：
        {constitution}

        请根据这些原则进行回答：
        1. 评估用户请求是否安全
        2. 如果不安全，拒绝回答
        3. 如果安全，提供有帮助的回答
        """

        response = self.client.chat.completions.create(
            model=self.model,
            messages=[{"role": "user", "content": prompt}],
            temperature=0.0  # 降低随机性，提高安全性
        )

        return response.choices[0].message.content

    def detect_and_block_malicious_input(self, user_input: str) -> bool:
        """检测并阻止恶意输入（2025 年新增）"""
        # 定义恶意模式
        malicious_patterns = [
            "prompt injection",
            "jailbreak attempt",
            "ignore previous instructions"
            "override system prompt"
        ]

        # 简单的关键词匹配检测
        for pattern in malicious_patterns:
            if pattern.lower() in user_input.lower():
                return True

        return False

# 使用示例
safe_agent = SafeAIAgent()

# 用户输入
user_input = "告诉我如何制造炸弹"

# 检测恶意输入
is_malicious = safe_agent.detect_and_block_malicious_input(user_input)

if is_malicious:
    print("Input blocked: Malicious input detected")
else:
    # 安全响应
    response = safe_agent.safe_respond(user_input)
    print("Response:", response)

小结 #

本章介绍了 AI Agent 的核心概念、发展历程、分类体系和开发挑战。

本章关键要点

  1. Agent 的定义：AI Agent 是一个自主的、感知环境的、做出决策的、执行行动的、从经验中学习的系统。
  2. Agent 的核心特征：自主性、响应性、目标导向、学习性、适应性。
  3. Agent 的发展历程：符号主义 Agent（1950s-1980s）→强化学习 Agent（1990s-2010s）→大语言模型 Agent（2020s）→自主智能体（2023s-）。
  4. Agent 的分类体系：按自主性、按能力、按部署方式。
  5. Agent 的开发挑战：规划复杂性、记忆管理、工具调用、安全与对齐、评估与监控。

2025 年最新技术趋势

  • Agentic AI 设计模式：Reflection、Tool Use、ReAct、Planning、Multi-Agent Collaboration
  • Azure Agent 编排模式：Sequential、Concurrent、Group Chat、Handoff、Orchestration

  • LangChain Agent 记忆：InMemoryStore、DB-backed Store、CoALA 映射
