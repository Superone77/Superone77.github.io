# Qwen3.5量化研究
本文发布于2026年3月13日，核心围绕Qwen3.5大模型的量化展开研究，对比了BF16、FP8、INT4、NVFP4多种量化格式的表现，给出了基于AutoRound的INT4和NVFP4量化实操方法，并通过基准测试明确了不同参数量Qwen3.5模型的量化适配性与避坑点，研究算力由欧洲AI云服务商Verda提供。

## 一、研究背景与核心价值
1. Qwen3.5模型参数覆盖0.8B-397B，可适配不同硬件和预算，但现代量化技术让低位量化模型具备更高性价比：4位量化的大参数量模型（如27B INT4）在内存接近小参数量模型（如9B）的情况下，性能显著更强。
2. 模型部分层对量化高度敏感，若保留其高精度，名义上的4位模型内存占用可能接近FP8模型（如Qwen3.5 27B GPTQ Int4与FP8内存分别为30.3GB、30.9GB）。
3. 本文不仅对比量化格式的精度，还结合内存占用分析权衡关系，同时提供量化实操方案和模型层的量化选择建议。

## 二、LLM量化与评估的软硬件准备
### （一）GPU选择
1. 量化本身成本低，AutoRound的逐块量化+`low_cpu_mem_usage`模式可降低峰值内存，无需GPU一次性容纳整个模型。
2. 模型评估对GPU内存带宽要求高，**B200**为优选（比H200显存多33%、带宽近乎翻倍，且专为FP4计算优化），作者研究使用了1块RTX Pro 6000、多块H200和1块B200（B200目前供应短缺）。
3. Verda云平台可便捷启动GPU实例，基础环境自带screen、vim等工具，仅需补充少量依赖。

### （二）软件安装
1. 必须安装`uv`（可选）和`python3-dev`（支撑vLLM的FlashInfer），且按**AutoRound→vLLM→Transformers**的顺序安装，避免vLLM替换不兼容的Transformers版本。
2. 额外安装`flash-linear-attention`和`causal-conv1d`可使AutoRound量化速度提升近一倍。
3. 待Qwen3.5支持稳定后，可直接通过`uv pip install auto-round vllm`完成核心依赖安装。

## 三、基于AutoRound的Qwen3.5量化实操
AutoRound是成熟的低位量化框架，CLI界面简洁，本文主要介绍INT4、NVFP4两种量化格式的操作方法，以及MoE模型的量化注意点。
### （一）INT4量化
1. 以Qwen3.5 9B为例，核心命令为`auto-round-best`，默认采用**W4A16**方案（权重4位，推理激活值16位），该方案适配主流推理框架。
2. 可选`--enable_torch_compile`参数，可使调优速度提升约25%（对Qwen3.5有效）；若追求量化速度，可替换为`auto-round`（牺牲部分精度）或`auto-round-light`（适合试验）。
3. 默认将视觉塔、LM head、归一化层、嵌入层保留为16位，可通过`--ignore_layers`排除易受量化影响的层（如linear_attn），提升模型精度（但会增加模型体积）。

### （二）NVFP4量化
1. 核心是将INT4命令的`scheme`替换为`NVFP4`，同时需显式指定`--format "llm_compressor"`（AutoRound默认NVFP4格式不兼容主流推理引擎）。
2. `torch_compile`与NVFP4量化路径不兼容，即使设置该参数也会被AutoRound忽略。
3. 必须通过`--ignore_layers`显式指定保留16位的模块，否则生成的配置文件会引发vLLM报错；若仍报错，可删除`quantization_config.json`并在`config.json`中配置未量化层的正则表达式。

### （三）MoE模型量化
1. 量化流程与普通模型一致，**建议将`shared_expert`层保留为16位**（英特尔官方默认选择）。
2. 量化`shared_expert`层对模型瘦身效果微乎其微，因该层仅占模型极小部分。

## 四、量化Qwen3.5的基准测试
测试覆盖Qwen3.5 9B、35B-A3B、27B，将最大序列长度限制为32k token（降低评估成本），因工具版本不匹配，未统计模型推理效率；模型精度主要由**量化算法、数据类型、精度布局**三个因素决定。
### （一）通用测试说明
对比了通义千问官方FP8/GPTQ INT4、英特尔INT4、CyanKiwi AWQ、各提供商NVFP4及作者基于AutoRound生成的INT4/NVFP4模型。

### （二）各模型测试结果
1. **Qwen3.5 9B**
    - 禁用思考功能：量化模型精度与原模型差距小，作者的AutoRound模型与英特尔版本精度持平。
    - 启用思考功能：量化模型精度最高下降33%且稳定性差，因量化模型“思考”更多，导致答案截断率翻倍；将线性注意力层保留为16位仅轻微改善，而推理需求低的基准（如MMLU Pro）量化后精度基本不变。
2. **Qwen3.5 35B-A3B（MoE模型）**
    - 启用思考功能：推理行为少于9B，答案截断率低；官方INT4模型（未量化注意力层）精度近乎持平原模型，FP8效果相近但内存更高；作者的AutoRound模型因量化了`shared_expert`，性能显著逊于英特尔版本。
    - 禁用思考功能：模型在H200上会无限生成内容，暂未找到原因；NVFP4格式对该MoE模型支持不完善，在B200上会生成无意义内容。
3. **Qwen3.5 27B**
    - 对量化适应性极强，量化后虽思考更多，但整体表现稳定。
    - 启用思考功能：**避免使用量化线性注意力层的NVFP4模型**，其负面影响会显现。
    - 禁用思考功能：量化线性注意力层无负面影响，不同NVFP4版本表现近乎一致；FP8版本内存更大且无性能优势，未测试官方GPTQ INT4（内存接近FP8）。

## 五、研究关键结论与未展示内容
1. 核心结论
    - MoE模型切勿量化`shared_expert`层，8位量化的可行性尚未验证。
    - 线性注意力层的量化负面影响**仅在生成长序列时显现**，短序列场景下无影响。
    - Qwen3.5 35B-A3B优先选择**官方INT4量化版本**（未量化注意力层），性价比最高。
    - Qwen3.5 27B量化需避开**量化线性注意力的NVFP4模型**和**FP8模型**。
    - 提升最大序列长度或可显著改善量化模型在推理类基准测试中的精度。
2. 未展示内容：Qwen3.5 27B NVFP4与397B模型的对比分析为付费内容，本文未披露。
